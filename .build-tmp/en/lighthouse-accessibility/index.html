Accessibility audits

Can all users access content and navigate your site effectively?

Overview

These checks highlight opportunities to improve the accessibility of your web app. Only a subset of accessibility issues can be automatically detected so manual testing is also encouraged.

What you'll learn

  • ARIA
  • Audio and video
  • Best practices
  • Contrast
  • Internationalization and localization
  • Names and labels
  • Navigation
  • Tables and lists
  • Additional items to manually check

ARIA #

  • Ensure ARIA attributes are allowed for an element's role
  • Ensure ARIA roles have required states and properties
  • Ensure parent roles contain required child role(s)
  • Ensure child roles are contained within required parent roles
  • Ensure ARIA role values are valid
  • Ensure ARIA attributes have valid values
  • Ensure ARIA attributes are valid and not misspelled or non-existent

Audio and video #

  • Ensure audio elements have captions
  • Ensure video elements have captions
  • Ensure video elements have audio descriptions

Best practices #

  • Ensure every ID attribute value is unique
  • Ensure timed refreshes do not exist
  • Ensure the viewport doesn't disable text scaling and zooming

Contrast #

  • Ensure background and foreground colors have a sufficient contrast ratio

Internationalization and localization #

  • Ensure every HTML document has a lang attribute
  • Ensure the lang attribute of the html element has a valid value
  • Ensure all lang attributes have a valid value

Names and labels #

  • Ensure buttons have discernible text
  • Ensure that each HTML document contains a title
  • Ensure IFrame and frame elements contain a non-empty title attribute
  • Ensure img elements have alternate text or a role of none or presentation
  • Ensure input types with a value "image" have alternate text
  • Ensure every form element has a label
  • Ensure links have discernible text
  • Ensure object elements have alternative text

Navigation #

  • Ensure accesskey values are unique
  • Ensure each page has at least one mechanism for a user to bypass navigation and jump straight to the content
  • Ensure tabindex attribute values are not greater than 0

Tables and lists #

  • Ensure dl elements are structured correctly
  • Ensure all child dd and dt elements have a dl parent
  • Ensure layout tables do not use data table elements
  • Ensure lists are structured correctly
  • Ensure list items are contained within a parent list
  • Ensure there's only one table header per table column
  • Ensure each table header has data cells

Additional items to manually check #

  • Manually check all custom interactive controls are keyboard focusable
  • Manually check all custom controls have appropriate ARIA roles
  • Manually check users can't accidentally trap their focus
  • Manually check custom interactive controls are keyboard focusable and display a focus indicator
  • Manually check that headings don't skip levels
  • Manually check interactive elements indicate their purpose and state
  • Manually check the tab order follows the DOM order
  • Manually check user's focus is directed to new content
  • Manually check offscreen content is hidden from assistive technology
  • Manually check landmark elements improve navigation
  • Manually check the visual layout of the page matches the DOM
